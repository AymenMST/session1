La programmation orienté objet
C’est quoi un Objet ?

L’objet est une entité logicielle qui représente la brique de base de la POO.

L’objet se caractérise par :

  1. Une identité (référence ou handle).
  2. Un état.
  3. Un comportement.

Exemple

Dans le monde réel, un objet peut être: un étudiant, un enseignant, un téléphone, une voiture, etc.

Exemple:Une voiture

  • le référence : voiture1 .
  • l'état: Nom, Marque, type, nombre de vitesses, Année, couleur,….
  • le comportement:rouler, tourner, accélérer, changer de vitesse, freiner,….
Cycle de vie d’objet

Un objet en programmation orienté objet (POO) a un cycle de vie de 3 phases :

  • Construction (en mémoire).
  • Utilisation (changements d’état par affectations, comportements par execution de méthodes).
  • Destruction

La classe

La classe est une description d’une famille d’objets ayant une même structure et un même comportement. Elle est caractérisée par :

  • Un nom.
  • Une composante statique.
  • Une composante dynamique.

Classe et objet

La classe est la machine qui fabrique les objets. Elle peut être considérée comme un moule ou un modèle à partir duquel on peut créer des objets.

Des objets créés à partir de la même classe auront des aspects semblables (mêmes attributs et même méthodes).

Concepts de base de la POO

La Programmation orienté Objet (POO) est basée sur les concepts suivants:

  • La composition.
  • L’encapsulation.
  • L’héritage.
  • Le polymorphisme.

La notion de classe

La notion de classe

La classe sert simplement à décrire la structure des objets (leurs attributs et leurs méthodes).

Le modèle général de définition d’une classe en Java est : 

 class Point {
/** instructions de définition des champs et des méthodes
 de la classe*/
		       }

Une classe ne peut pas être utilisée pour executer des programmes. Pour cela il faut l’instancier (créer des objets).

Exemple

  • Supposons qu’un point est représenté par deux coordonnées entières: abscisse X et ordonnée Y.
  • Supposons que nous souhaitons pouvoir:
    • Initialiser les coordonnées d’un point.
    • Déplacer ce point dans le plan.
    • Faire une rotation de ce point.
    • Afficher les coordonnées de ce point
Les Attributs
  • Les attributs sont les données de la classe
  • Un attribut peut être:
    • de type élémentaire: entier, réel (float, double), booléen, caractère.
    • de type objet. 
 class Point {
   float abscisse,ordonnée;// les attributs de la class point

		       }
Les méthodes
  • Une méthode est un regroupement d’instructions semblable aux fonctions et procédures sauf qu’elle s’exécute toujours sur un objet précis. 
     class Point {
  float abscisse,ordonnée;
	void initialiser ( float x , float y) {
		abscisse = x;
		ordonnée = y;
		} //une méthode de la class point
		       }

Les constructeurs
  • Un constructeur est une méthode spéciale utilisée pour crée un nouvel objet. Le constructeur se caractérise par:
    • Il a le même nom que celui de la classe.
    • Ne retourne aucune valeur.
    • N’a aucun type de retour, même pas void.
     class Point {
  float abscisse,ordonnée;
	Point ( float x , float y) {
		abscisse = x;
		ordonnée = y;
		} //le constructeur de la class point
		       }

Les modificateurs d'accès
  • indiquent si les autres classes de l'application pourront accéder ou non à la classe,méthode...
    • public : toutes les classes peuvent accéder à l'item
    • protected : seules les classes dérivées et les classes du même package peuvent accéder à l'item.
    • private : l'item est seulement accessible depuis l'intérieur de la classe où il est défini.
    • (par défaut) : seules les classes du même package peuvent accéder à l'item.
L’instanciation
  • L’instanciation est la concrétisation d’une classe en un objet. Pour créer un emplacement pour l’objet lui-même, il faut utiliser le mot clé new.
  • Exemple : 
     Point centre = new Point(5.0f,3.5f);
La composition
  • La composition en java est la technique de conception permettant d'implémenter une relation a dans des classes.
  • Exemple : 
     public class Circle{
Int rayon;
Point centre;
}
L’encapsulation
  • Elle consiste à regrouper des données (attributs) et un comportement (méthodes) dans une même classe et à réglementer l’accès aux données de l’extérieur (par d’autres objets).
  • Les modificateurs d'accès et les setters et les getters en Java garantissent ce principe.
L’encapsulation
  • Exemple :
 protected class Circle{
private int rayon;
private Point centre;
public void setRayon(int r){
rayon =r;
}
public int getRayon(){
return rayon ;
}
L’héritage
  • En Java, ce mécanisme est mis en œuvre au moyen du mot-clé extends
  • Exemple : 
 class Circle1 extends point{
int rayon;
	Circle1 (int r){
	super();
	rayon=r;
	}
}
  • Le mot clé super permet d'accéder aux membres de la super-classe d'une classe.
    • Les packages
    • Le package en Java est un mécanisme pour encapsuler un groupe de classes, sous-packages et interfaces.
      • Prévenir les conflits de noms
      • Fournir un accès contrôlé
      • Faciliter la recherche / la localisation et l'utilisation
      • Les packages peuvent être considérés comme une encapsulation de données
